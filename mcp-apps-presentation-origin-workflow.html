The MCP APP solution

Renders an interactive UI inside the conversation

Key Capabilities

  • Context switching elimination

    Elimination of "context switching" by anchoring functional tools directly within the flow of a conversation.

  • Bidirectional data flow

    Apps use existing MCP patterns for bidirectional communication, eliminating the need for custom APIs, authentication, or complex state management.

  • Security guarentees

    Hosts can safely render third-party apps.

MCP Apps: Security & Isolation Model

  • Strong Iframe Sandboxing

    All UI components run in a strictly isolated environment. This "glass wall" prevents the app from accessing the host's DOM, cookies, local storage, or executing unauthorized scripts.

  • Auditable Communication

    The app and host communicate exclusively via the postMessage API using JSON-RPC. Every interaction is structured, loggable, and can be inspected for suspicious activity.

  • Granular Host Control

    The host acts as a gatekeeper, with the power to restrict specific capabilities (e.g., disabling link opening or limiting which tools the app can call).

  • Pre-declared Templates

    Hosts can review and vet HTML content before it is even rendered, allowing for the proactive blocking of malicious code.

  • User Consent & Transparency

    Sensitive actions—such as UI-initiated tool calls—require explicit user approval, ensuring that no critical operations happen without human oversight.

When to use MCP Apps

Data Exploration

It’s about transforming the AI from a simple "messenger" into a fully functional analytics cockpit where the user can manipulate data visually and intuitively.

Configuration Wizards

It’s about shifting from a "chat-based" configuration to a "structured, dynamic form" approach to increase efficiency and reduce errors in complex deployments.

Document Review

It is about "Direct Manipulation," where the AI workspace becomes a functional dashboard, allowing users to interact with files and media natively rather than just talking about them.

Real-time Monitoring

It is about "Live Observability," where the system provides a constant pulse of information, allowing users to monitor health and performance without any extra effort.

Multi-step Workflows

It is about "Workflow Automation," turning a conversational AI into a structured productivity tool that handles high-volume, repetitive tasks through a dedicated, stateful interface.
